Partner API

Embed Atomic's cross-chain swaps in your product. Non-custodial — we return a quote plus a signable transaction; your user signs and broadcasts it. You earn on every swap.

Authentication

Send your API key as a Bearer token on every request. Get a key from the partner dashboard — it's shown once, so store it securely (we keep only its hash).

Authorization: Bearer ak_live_xxxxxxxxxxxxxxxxxxxxxxxx

Base URL: https://atomicpay.cloud · Rate limit: 120 req/min per key.

Endpoints

GET /v1/partner/assets

Assets certified for live routing.

{ "assets": [{ "assetId": "BASE.USDC", "symbol": "USDC", "chain": "BASE", "decimals": 6 }], "count": 15 }

POST /v1/partner/quote

Create a swap quote. Amounts are atomic integer strings (respect each asset's decimals).

{
  "fromAsset": "BASE.USDC",
  "toAsset":   "ETH.USDC",
  "amount":    "5000000",     // 5 USDC @ 6 decimals
  "userAddress": "0x…",       // destination (funds land here)
  "fromAddress": "0x…"        // source (signs); optional same-chain
}

Response 201 (status QUOTED):

{
  "quote": {
    "id": "…", "status": "QUOTED",
    "estimatedOutputAmount": "4862500", "expiresAt": "…",
    "execution": {
      "transactionRequest": { "to": "0x1231DEB6…", "data": "0x…", "value": "0x0", "chainId": 8453 },
      "approvalAddress": "0x1231DEB6…"   // ERC20: approve this spender first
    }
  },
  "fee": { "totalBps": 285, "partnerBps": 85, "platformBps": 200, "markupBps": 35 }
}

Codes: 201 QUOTED · 403 sanctioned/blocked · 409 price-impact halt · 400 bad request.

GET /v1/partner/quote/:id

Current status of one of your quotes.

GET /v1/partner/statements · /earnings

Earnings rolled up by month & asset (statements) or per-swap (earnings) — your settlement records.

Integration flow

  1. GET /assets → choose the pair.
  2. POST /quote → get execution.transactionRequest.
  3. ERC20 source: if allowance to execution.approvalAddress is low, have the user send an approve first and wait for it to confirm.
  4. Have the user sign & broadcast execution.transactionRequest.
  5. Track status via a webhook (below) or by polling /quote/:id.

Webhooks

Set a webhook URL in the dashboard to receive a signed POST when a swap changes state — no polling.

Events: swap.authorized, swap.broadcast, swap.updated, swap.completed.

POST https://your-app.com/atomic-webhook
x-atomic-signature: <hex hmac>

{ "type": "swap.completed", "quoteId": "…", "status": "COMPLETE", "sentAt": "…" }
Verify every webhook. Compute HMAC-SHA256(rawBody, yourSigningSecret) and constant-time compare it to the x-atomic-signature header. Reject on mismatch.

Notes

  • Non-custodial — Atomic never holds funds; the user's wallet signs & broadcasts.
  • Compliance — source/destination addresses are screened (OFAC + on-chain oracle); sanctioned parties return 403.
  • Quotes expire at expiresAt (default 90s) — re-quote if it lapses.
